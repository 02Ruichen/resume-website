# 求职全流程工作流（Job Application Workflow）

> 已集成简历网站 `D:\claude\简历\resume-website\`，Phase 3 后可直接更新 `resume-data.json` 并构建部署。

---

## 触发条件与适用场景

用户提供一个 **JD**（文本/图片/链接）+ **简历**（PDF/文本），希望完成从 JD 分析到面试准备的完整求职流程时触发。

适用场景：
- 批量投递时每个新 JD 走一遍相同流程
- 首次系统性准备面试
- 需要快速匹配新岗位要求

---

执行前**必须**获取两份输入：

| 输入 | 格式 | 说明 |

| JD | 文本 / 图片 / URL | 目标岗位完整描述 |
| 简历 | PDF / 文本 | 用户当前完整简历 |

> ⚠️ 如果用户只提供了其中一份，**主动索要另一份**。两份齐全后按 Phase 1→6 顺序执行。

## 工作流步骤

### Phase 1: JD 高频能力拆解

> 从 JD 中提取结构化能力清单，作为后续所有 Phase 的输入基准。

**执行步骤：**

1. 解析 JD，提取所有能力要求（职责描述 + 任职资格 + 隐含要求）
2. 分类为三维度：
   - **硬技能**：工具、技术、专业能力、业务领域知识
   - **软技能**：通用素质、工作方法、沟通协作能力
   - **核心诉求**：岗位核心工作目标、招聘方真正想找的人画像
3. 合并重复项，按出现频次从高到低排序
4. 标注优先级：高频（出现 ≥2 次或单独强调）、中频、低频

**输出格式：**

```
岗位概况
公司名 · 岗位名 · 城市 · 薪资 · 经验要求 · 学历

硬技能（按优先级排序）
优先级    能力项        JD 出处
高频       xxx          职责1、资格3
中频       xxx          职责2

软技能（按优先级排序）
（同上格式）

核心诉求总结
一段话概括：这个岗位本质上在找什么样的人？
```

---

### Phase 2: 项目经历 STAR 重构

> 基于 Phase 1 能力表，重构 2-4 段与目标岗位高度匹配的项目经历。

**执行步骤：**

1. 从用户原始简历中筛选可重构的项目（优先选与 JD 最相关的）
2. 用 **STAR 法则**重写每段项目：
   - **S**（Situation）：项目背景和业务场景，1-2 句
   - **T**（Task）：你的角色和核心目标，1 句
   - **A**（Action）：你具体做了什么，3-5 个动作，植入 JD 关键词
   - **R**（Result）：量化结果，**必须有数据**（转化率、增长率、效率提升等）
3. 每段项目必须：
   - **四字小标题**（如"转化攻坚""出海拓荒"）
   - 植入 **2-3 个** Phase 1 高频关键词（自然融入，不堆砌）
   - 突出**可量化落地结果**
4. 语言风格：大厂简历风格，简洁有力，避免虚词

**关键词植入检查：** 完成后对照 Phase 1 高频能力表，确认每个高频关键词至少在一段项目中出现。

---

### Phase 3: 简历优化

> 四维优化，输出可直接投递的优化版简历。

**四维优化：**

**1. 个人简介（3 句话）**
- 第 1 句：经验年限 + 专注方向
- 第 2 句：最亮的量化成果（从 Phase 2 项目中提炼）
- 第 3 句：与目标岗位核心诉求的匹配点

**2. 技能栏**
- 按 Phase 1 硬技能优先级排序
- 岗位刚需技能放最前，弱相关技能精简或删除

**3. 项目经历**
- 替换为 Phase 2 重构后的项目
- 排序原则：与目标岗位匹配度最高的项目放第一位

**4. 关键词植入**
- 全文检查 Phase 1 高频关键词覆盖率
- 精简无关内容（与目标岗位无关的经历压缩或删除）
- 整体风格简洁专业，适配大厂投递规范

**输出：** `优化版简历.md`（Markdown 格式，可直接投递）。

---

### Phase 4: 面试准备

> 准备 1min30s 自我介绍 + 结构化面试题库。

#### 4a. 自我介绍（1min30s 以内，200-250 字）

**三段式结构：**

| 段落 | 时长 | 内容 |
|------|------|------|
| 第一段 · 身份锚定 | 15s | 个人信息 + 面试岗位，一句话定位核心标签 |
| 第二段 · 供需匹配 | 60s | ⭐核心段，逐条对齐 JD 高频能力，每个能力点用"岗位要求→我的经历→数据结果"句式 |
| 第三段 · 表达意愿 | 15s | 对岗位/团队的兴趣 + 信心和期待 |

**校验标准：** 朗读计时 ≤ 1min30s，每段都有实质内容不空泛。

#### 4b. 结构化面试题（总分总框架）

**答案结构：**
- **总** · 结论先行（1句）：直接给出核心观点，不铺垫
- **分** · 展开论述（3 个要点）：用"第一/第二/第三"分层，每点 = 观点 + 举例或数据
- **总** · 收束总结（1句）：回扣结论 + 表态或展望

**三类题目，每类 4-5 题：**

| 类别 | 覆盖方向 | 建议时长 |
|------|----------|----------|
| 项目深挖类 | 难点、技术选型、复盘、踩坑、归因分析 | 90-120s/题 |
| 专业高频类 | 产品方法论、数据思维、用户洞察、行业理解 | 60-90s/题 |
| 行为类 | 团队协作、压力管理、冲突处理、职业规划 | 60s/题 |

**题目生成规则：**
- 项目深挖类 → 基于 Phase 2 重构项目，模拟面试官追问
- 专业高频类 → 基于 Phase 1 高频能力表，生成该岗位方向常见题
- 行为类 → 结合目标公司文化和岗位特点定制

> 每题附完整示范答案（总分总结构），标注建议答题时长。

---

### Phase 5: 可复用 SOP 输出

> 将 Phase 1-4 的完整流程沉淀为标准化操作清单。

**输出内容：**

1. **5 Phase 操作步骤清单**：每个 Phase 的输入、操作、输出、耗时预估
2. **投递前 Checklist**：
   - JD 能力清单已拆解，高频词已标记
   - 至少 2 段项目已用 STAR 重构，植入高频关键词
   - 简历个人简介已针对目标岗位定制
   - 技能栏按 JD 优先级排序
   - 全文关键词覆盖率 > 80%
   - 自我介绍已准备，朗读 < 1min30s
   - 面试题至少 12 道已准备，含示范答案
   - 无错别字、格式统一、排版整洁

---

### Phase 6: 同步到简历网站 ⭐

> 将 Phase 3 优化的简历内容写入 `resume-data.json`，构建部署 + 导出 PDF。

**1. 判断目标版本**
- 投递互联网/AI 公司 → 更新 `internet` 版本
- 投递国企/央企/政府 → 更新 `gov` 版本
- 不确定时问用户："这个岗位你想更新互联网版还是国企版？"

**2. 映射优化内容到 resume-data.json**
- `targetPositions`：从 Phase 1 岗位名提取意向岗位
- 实习经历 `details`：从 Phase 2 STAR 重构项目映射（每条 ≤115字/<b>加粗，3条合计 ≤285字）
- 实习经历 `keywords`：核心能力体现，≤35 字
- `skills[].items`：从 Phase 3 技能栏映射，每类 ≤120 字
- `selfEvaluation`：从 Phase 3 个人简介提炼

**3. 验证 + 构建 + 部署**
```bash
cd D:\claude\简历\resume-website
python validate_layout.py    # 排版验证（必须通过）
python build.py              # 构建 + 部署到 GitHub Pages
```

**4. 确认结果**
- ✅ 验证通过 + 部署 200 → 告知用户线上地址，打开本地文件预览
- ⚠️ 验证失败 → 根据报错精简文字，重新验证
- ⚠️ 部署非 200 → 告知用户网络问题，本地文件仍可用

**5. 导出投递用 PDF**
- 打开 `index.html` → 切换对应版本 → 点击「📄 导出PDF」
- 文件名格式：`MMDD-版本名-刘芮辰.pdf`（如 `0718-互联网-刘芮辰.pdf`）
